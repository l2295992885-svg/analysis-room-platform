# 综合分析室数据分析平台

本仓库用于建设“综合分析室数据分析平台”。系统定位为内部中后台管理平台，后续将包含违章管理、LKJ 分析、音视频分析、单兵分析、解锁统计、调度命令、施工作业、试运机车、月度报表、统计分析、基础数据、系统管理等模块。

## 第一阶段目标

第一阶段只实现一个样板业务闭环：

- 违章管理 > 每日 LKJ 音视频违标公示

该样板业务需要覆盖：分析员录入、Excel 导入、编码校验、表格预览、班长审核、主任审核、下发车间、车队、指导组确认、不属实反馈、附件上传、返回复核、结果库入库、日志证据链、Excel 导出、信箱中心。

## 初始技术路线

- 前端：参考 Art Design Pro 的 UI 风格、布局、表格、表单和常用组件。
- 后端：Spring Boot。
- 数据库：第一版使用 MySQL 8.x。
- 权限：参考若依成熟的用户、角色、部门、菜单、按钮权限、数据权限、安全拦截、登录日志、操作日志设计。
- 架构：第一版不使用微服务，不引入复杂工作流引擎，不做大屏。

## 开发原则

- 前端权限只负责显示，后端接口必须再次校验权限。
- 所有业务查询必须考虑数据权限。
- 所有导出必须校验导出权限并记录日志。
- 所有提交、审核、退回、修改、反馈、上传、入库操作必须写业务证据链日志。
- 不允许物理删除历史业务数据。
- 不允许覆盖历史违标编码快照。

## 目录规划

```text
frontend/   前端工程，采用 Art Design Pro 风格
backend/    后端工程，采用 Spring Boot
database/   MySQL 初始化脚本和后续迁移脚本
docs/       项目规则、权限规则、流程规则、日志规则、导入导出规则
scripts/    本地启动、构建、部署辅助脚本
```

详细开发规则见 `AGENTS.md` 和 `docs/` 目录。
