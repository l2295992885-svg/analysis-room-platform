# 流程规则

## 设计原则

每日 LKJ 音视频违标公示是第一阶段样板功能，但流程能力要按平台公共能力设计，后续可复用于其他模块。

## 核心流程

```text
草稿
→ 分析员已提交
→ 班长待审核
→ 主任待审核
→ 主任已下发车间
→ 车间待确认
→ 车队待确认
→ 指导组待确认
→ 指导组确认无误 / 指导组反馈不属实
→ 逐级返回
→ 分析室复核
→ 主任最终确认
→ 已入结果库 / 已撤销不计入
```

## 流程控制原则

1. 记录只能按允许的流程节点流转。
2. 不能通过前端绕过后端流程状态校验。
3. 每次流转都必须生成待办任务和业务日志。
4. 每次退回必须填写原因。
5. 不属实反馈必须填写原因并支持附件。
6. 主任最终确认后才能进入结果库。
7. 没有日志证据链的数据不得进入结果库。

## 信箱与待办关系

- 待办列表：当前需要我处理的业务任务。
- 信箱中心：所有流转消息，包括收件箱、我提交的、退回给我的、反馈给我的、我反馈的、抄送给我的、已处理。

## 公共流程表建议

- `biz_task`：当前待办任务。
- `biz_message`：信箱消息。
- `biz_action_log`：业务动作日志。
- `biz_action_log_detail`：字段变更明细。
- `biz_flow_config`：后续流程配置预留。
- `biz_flow_node`：后续流程节点预留。
