# 日志证据链规则

## 日志目标

本项目日志不是单纯技术日志，而是业务证据链。业务人员应能看懂：几月几号、谁、对哪条记录、做了什么、改了什么、退回了什么、反馈了什么、导出了什么。

## 必须记录的动作

- 新增
- 保存草稿
- Excel 导入
- 预览确认
- 提交
- 审核通过
- 退回修改
- 下发
- 确认无误
- 反馈不属实
- 上传附件
- 查看附件
- 修改字段
- 返回复核
- 最终确认
- 入结果库
- 撤销不计入
- 导出 Excel

## 日志字段建议

- 日志 ID
- 业务类型
- 业务记录 ID
- 操作人 ID
- 操作人姓名
- 操作人角色
- 操作部门 ID
- 操作类型
- 操作说明
- 操作前状态
- 操作后状态
- 操作时间
- IP 地址
- 附件数量
- 关联批次 ID

## 字段变更日志

凡是修改业务关键字段，应记录：

- 字段名
- 修改前值
- 修改后值
- 修改人
- 修改时间
- 修改原因

## 结果库要求

没有完整业务证据链的数据，不允许进入最终结果数据库。
