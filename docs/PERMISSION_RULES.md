# 权限规则

## 权限分层

本项目权限分为四类：

1. 功能权限：能否进入某个菜单或页面。
2. 操作权限：能否新增、修改、审核、退回、下发、反馈、导出、入库。
3. 数据权限：能查看哪些组织、人员、批次、流程记录。
4. 字段权限：某些关键字段能否编辑。

## 参考模型

系统权限参考若依成熟设计：

- 用户管理
- 角色管理
- 部门管理
- 菜单管理
- 按钮权限
- 角色菜单权限
- 角色部门数据权限
- 登录日志
- 系统操作日志

业务层在此基础上增加：

- 分析员-班长绑定
- 车间-车队-指导组关系
- 业务数据范围配置
- 导出权限配置
- 附件查看权限配置
- 流程节点操作权限

## 前端权限和后端权限

前端权限只负责显示：

- 菜单是否显示
- 页面是否能进入
- 按钮是否显示

后端权限负责真正安全：

- 接口是否允许调用
- 当前用户是否有当前记录的数据权限
- 当前记录是否处于允许操作的流程状态
- 当前用户是否有导出、附件查看、最终入库等特殊权限

## 角色初始规划

- 分析员
- 分析班长
- 分析室主任
- 车间
- 车队
- 指导组
- 系统管理员

## 数据权限原则

- 分析员默认只能看本人录入、本人待处理、本人被退回的数据。
- 班长默认看所属分析员提交的数据。
- 主任默认看分析室相关数据和责任部门返回数据。
- 车间默认看下发到本车间的数据。
- 车队默认看下发到本车队的数据。
- 指导组默认只看本指导组待确认和已处理数据。
- 系统管理员管理系统配置，不等于可以随意修改业务历史数据。

## 导出权限

导出权限独立于查看权限。能看不代表能导出。

所有导出必须记录：

- 导出人
- 导出时间
- 导出页面
- 导出条件
- 导出数据量
- 导出文件名
