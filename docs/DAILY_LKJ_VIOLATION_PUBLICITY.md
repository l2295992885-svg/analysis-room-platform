# 每日 LKJ 音视频违标公示业务说明

> 文件用途：本文件供 Codex / AI 编程工具 / 开发人员理解并实现“违章管理 > 每日 LKJ 音视频违标公示”业务。实现该模块前，必须先阅读 `README.md`、`AGENTS.md`、`docs/PROJECT_BRIEF.md`、`docs/PERMISSION_RULES.md`、`docs/WORKFLOW_RULES.md`、`docs/LOG_RULES.md`、`docs/EXPORT_RULES.md`，再阅读本文件。

---

## 1. 模块定位

“每日 LKJ 音视频违标公示”是综合分析室数据分析平台第一阶段样板业务。

它不是孤立的录入页面，而是用于沉淀平台公共能力的核心样板，包括：

- 角色权限
- 数据权限
- 字段权限
- 信箱中心
- 待办任务
- 文件附件
- 聊天协同业务卡片
- Excel 导入导出
- 违标编码字典
- 业务日志证据链
- 结果库版本化
- 统计查询

任何实现不得把本模块写成不可复用的“单页面功能”。后续 LKJ 分析、音视频分析、单兵分析、调度命令、施工作业、试运机车、月度报表等模块都要复用本模块沉淀出的公共能力。

---

## 2. 核心业务对象

本模块的核心业务对象为：

```text
daily_violation_record
```

表示一条每日 LKJ / 音视频违标公示记录。

它可以来源于：

- 分析员手工录入
- Excel 批量导入
- 后续其他分析模块推送

一条记录从草稿开始，经过分析员提交、班长审核、主任审核、下发责任部门、指导组确认或反馈、分析室复核、主任最终确认，最终进入结果库。

## 3. 角色与职责

### 3.1 分析员

- 手工录入违标记录。
- Excel 导入违标记录。
- 维护本人草稿。
- 预览并提交记录。
- 处理被班长或主任退回的记录。
- 查看本人提交记录的流程状态。
- 查看与本人相关的反馈、不属实材料和处理结果。

默认数据范围：本人草稿、本人提交、退回给本人、本人参与复核的数据。

### 3.2 分析班长

- 审核所属分析员提交的记录。
- 审核通过后提交主任。
- 退回分析员修改。
- 查看所属分析员相关记录。
- 可根据权限导出本范围数据。

默认数据范围：所属分析员提交的数据和本流程相关数据。

### 3.3 分析室主任

- 审核班长提交的记录。
- 下发责任车间。
- 处理责任部门反馈和不属实争议。
- 可退回班长或分析员复核。
- 最终确认并入结果库。
- 可撤销不属实记录并标记不计入。
- 查看分析室业务范围内数据。

默认数据范围：分析室业务数据、待主任处理数据、结果库数据。

### 3.4 车间

- 接收主任下发到本车间的记录。
- 下发给对应车队。
- 汇总车队和指导组确认结果。
- 将确认结果或反馈逐级返回。

默认数据范围：下发到本车间的数据。

### 3.5 车队

- 接收车间下发数据。
- 下发给对应指导组。
- 汇总指导组确认结果或反馈。
- 将结果返回车间。

默认数据范围：下发到本车队的数据。

### 3.6 指导组

- 接收车队下发数据。
- 对记录确认无误，或反馈不属实。
- 不属实反馈必须填写原因并上传证据附件。

默认数据范围：下发到本指导组的数据。

### 3.7 系统管理员

- 管理用户、角色、部门、菜单、按钮权限、数据权限。
- 管理字典、模板、系统配置。
- 不等于可以绕过业务权限随意修改历史业务数据。
- 管理员修改业务数据必须有授权、原因和日志。

## 4. 主流程

### 4.1 标准流程

```text
分析员录入 / Excel 导入
→ 编码校验
→ 表格预览
→ 分析员提交
→ 班长审核
→ 主任审核
→ 主任下发车间
→ 车间下发车队
→ 车队下发指导组
→ 指导组确认无误
→ 车队汇总
→ 车间汇总
→ 主任最终确认
→ 入结果库
```

### 4.2 不属实反馈流程

```text
主任下发车间
→ 车间下发车队
→ 车队下发指导组
→ 指导组反馈不属实
→ 上传证据附件
→ 车队查看并返回
→ 车间查看并返回
→ 主任接收反馈
→ 主任处理：维持原处理 / 退回复核 / 认定不属实并撤销不计入
→ 如退回复核：班长 / 分析员复核
→ 重新提交主任
→ 主任最终确认
→ 入结果库或撤销不计入
```

### 4.3 退回复核流程

```text
班长退回分析员
或
主任退回班长 / 分析员
→ 接收人修改或补充说明
→ 重新提交
→ 继续审核流程
```

退回必须填写原因。

## 5. 状态设计

### 5.1 建议状态枚举

状态值应使用稳定英文编码，前端显示中文名称。

| 状态编码 | 中文名称 | 说明 |
|---|---|---|
| DRAFT | 草稿 | 分析员暂存，未提交 |
| IMPORT_PREVIEW | 导入预览 | Excel 导入后待确认 |
| ANALYST_SUBMITTED | 分析员已提交 | 已提交给班长 |
| LEADER_PENDING_REVIEW | 班长待审核 | 班长待处理 |
| LEADER_RETURNED | 班长已退回 | 退回分析员修改 |
| DIRECTOR_PENDING_REVIEW | 主任待审核 | 主任待处理 |
| DIRECTOR_RETURNED | 主任已退回 | 主任退回班长或分析员 |
| DIRECTOR_DISPATCHED_WORKSHOP | 主任已下发车间 | 已下发责任车间 |
| WORKSHOP_PENDING_DISPATCH | 车间待下发 | 车间待下发车队 |
| WORKSHOP_DISPATCHED_TEAM | 车间已下发车队 | 已流转到车队 |
| TEAM_PENDING_DISPATCH | 车队待下发 | 车队待下发指导组 |
| TEAM_DISPATCHED_GROUP | 车队已下发指导组 | 已流转到指导组 |
| GROUP_PENDING_CONFIRM | 指导组待确认 | 指导组待确认或反馈 |
| GROUP_CONFIRMED | 指导组确认无误 | 指导组确认 |
| GROUP_FEEDBACK_NOT_TRUE | 指导组反馈不属实 | 指导组提出争议 |
| DIRECTOR_RECHECK_PENDING | 主任待复核 | 反馈返回主任待处理 |
| ANALYST_RECHECK_PENDING | 分析员待复核 | 主任退回分析员复核 |
| LEADER_RECHECK_PENDING | 班长待复核 | 主任退回班长复核 |
| DIRECTOR_FINAL_PENDING | 主任待最终确认 | 复核完毕待主任最终确认 |
| FINAL_CONFIRMED | 主任最终确认 | 允许进入结果库 |
| RESULT_STORED | 已入结果库 | 已生成结果库记录 |
| CANCELLED_NOT_COUNTED | 已撤销不计入 | 认定不属实或撤销，不参与统计 |

### 5.2 状态流转要求

- 后端必须校验当前状态是否允许当前动作。
- 前端隐藏按钮不等于权限通过。
- 任何状态跳转必须写业务日志。
- 不允许通过接口直接跳过中间审核节点。
- 入结果库前必须检查是否存在完整业务日志链路。

- 扣分

- 内部复杂统计字段

### 6.3 日期规则

- Excel 导出标题可显示提报日期，但表格列只显示“违章发生日期”。

## 7. 违标编码规则

### 7.1 基本规则

- 违标编码必须来自 `violation_code_dict`。
- 不存在、停用、未生效、已过期的编码不得提交。
- 选择编码后自动带出：违标名称、性质、类别、类型、来源、版本。
- 历史记录必须保存编码快照。
- 字典后续修改不得覆盖历史记录快照。

### 7.2 版本规则

应根据 `violation_date` 匹配编码生效版本。

如果同一编码存在多个版本：

```text
按违章发生日期匹配 effective_date <= violation_date <= expiry_date 的版本
```

如果无法匹配：

- 阻止提交。
- 提示编码版本不存在或日期不在有效期内。

### 7.3 手动覆盖规则

原则上不允许手动修改由编码带出的性质、类别、类型。

如确需覆盖：

- 必须有特殊权限。
- 必须填写原因。
- 必须记录字段变更日志。

## 8. Excel 导入规则

### 8.1 导入流程

```text
上传 Excel
→ 保存原始文件
→ 创建导入批次
→ 保存每一行原始 JSON
→ 字段映射
→ 数据校验
→ 进入预览表
→ 用户勾选有效数据
→ 确定提交
→ 生成正式记录或草稿
```

### 8.2 导入批次必须记录

- 批次 ID
- 业务类型
- 模板编码
- 原文件名
- 文件路径
- 导入人
- 导入时间
- 总行数
- 成功行数
- 警告行数
- 失败行数
- 当前状态

### 8.3 原始行必须记录

每一行原始 Excel 数据必须保存原始 JSON 快照，便于追溯。

### 8.4 校验结果分类

| 类型 | 说明 | 是否允许提交 |
|---|---|---|
| 校验通过 | 字段完整且规则正确 | 允许 |
| 需要确认 | 疑似重复、部门模糊、人员变动等 | 用户确认后允许 |
| 校验失败 | 缺必填字段、编码不存在、人员不存在等 | 不允许 |

### 8.5 常见校验错误

- 违标编码不存在。
- 违标编码已停用。
- 编码与性质不匹配。
- 工号不存在。
- 工号与姓名不一致。
- 责任部门不存在。
- 违章日期为空。
- 拟考核内容为空。
- 疑似重复导入。

### 8.6 重复判断建议

可综合以下字段判断疑似重复：

- 违章发生日期
- 工号
- 违标编码
- 责任部门
- 拟考核内容摘要
- 车次 / 机车 / 地点

疑似重复不得自动删除，应提示用户确认。

## 9. 预览与提交规则

### 9.1 预览页

分析员录入或 Excel 导入后，必须先进入预览页。

预览页要求：

- 表格样式接近 Excel。
- 支持勾选 1-N 条记录。
- 支持显示校验结果。
- 支持修改未提交草稿。
- 支持导出预览数据。
- 支持取消导入批次。

### 9.2 确定提交

点击“确定提交”时，后端必须校验：

- 当前用户是否有提交权限。
- 所选记录是否属于当前用户可操作范围。
- 记录是否处于草稿或预览状态。
- 必填字段是否完整。
- 违标编码是否有效。
- 是否存在校验失败数据。

提交成功后：

- 记录状态进入 `ANALYST_SUBMITTED` 或 `LEADER_PENDING_REVIEW`。
- 生成班长待办。
- 生成信箱消息。
- 写业务日志。

## 10. 审核、下发与反馈动作

### 10.1 班长审核

可执行动作：

- 审核通过：提交主任。
- 退回修改：退回分析员。

必须写日志。

### 10.2 主任审核

可执行动作：

- 审核通过并下发车间。
- 退回班长。
- 退回分析员。
- 最终确认。
- 撤销不计入。

主任是最终业务确认角色。

### 10.3 车间处理

可执行动作：

- 下发车队。
- 汇总车队结果。
- 将反馈返回主任。

### 10.4 车队处理

可执行动作：

- 下发指导组。
- 汇总指导组结果。
- 将反馈返回车间。

### 10.5 指导组确认

可执行动作：

- 确认无误。
- 反馈不属实。

反馈不属实时必须填写原因和说明，可上传附件。

## 11. 不属实反馈规则

### 11.1 反馈原因类型

建议原因类型：

- 责任人不符
- 工号不符
- 时间不符
- 地点不符
- 机车 / 车次不符
- 违标编码不符
- 事实描述不符
- 重复提报
- 其他

### 11.2 反馈表单字段

- 记录 ID
- 反馈类型
- 原因类型
- 详细说明
- 附件列表
- 反馈人
- 反馈部门
- 反馈时间

### 11.3 反馈处理

主任处理不属实反馈时可选择：

- 维持原处理。
- 退回复核。
- 认定不属实并撤销不计入。
- 要求补充材料。

指导组反馈不能直接删除原记录。

## 12. 附件与文件中心规则

本模块附件必须接入平台文件中心。

附件场景包括：

- Excel 原始导入文件
- 不属实反馈证据
- 分析员补充材料
- 主任处理依据
- 导出归档文件

附件必须记录：

- 文件 ID
- 业务类型
- 业务 ID
- 关联动作日志 ID
- 原文件名
- 存储路径
- 文件类型
- 文件大小
- 上传人
- 上传时间
- 权限范围

安全规则：

- 看到业务记录不一定能下载附件。
- 下载附件必须校验权限。
- 查看、下载、转发、删除附件必须留痕。
- 删除附件原则上逻辑删除，不物理删除。

## 13. 信箱、待办与聊天协同

### 13.1 待办任务

每次流程流转，应生成或关闭对应待办。

待办只表示“当前需要我处理的任务”。

典型待办：

- 班长待审核
- 主任待审核
- 车间待下发
- 车队待下发
- 指导组待确认
- 主任待复核
- 分析员待复核

### 13.2 信箱消息

信箱用于保存业务流转消息。

常见分类：

- 收件箱
- 我提交的
- 退回给我的
- 反馈给我的
- 我反馈的
- 抄送给我的
- 已处理

信箱消息应绑定业务类型和业务 ID。

### 13.3 聊天业务卡片

聊天协同可以发送业务卡片，但不得替代流程流转。

业务卡片字段建议：

- card_id
- message_id
- business_type
- business_id
- business_title
- business_summary
- current_status
- target_action
- jump_url
- creator_id
- create_time

安全规则：

- 别人发来的业务卡片不代表当前用户有权限查看。
- 点击卡片必须重新请求后端。
- 后端必须校验功能权限、操作权限、数据权限、流程状态权限。
- 聊天提醒不等于审核通过。
- 草稿分享不等于正式提交。

| violation:daily:dispatch:team | 下发车队 |
| violation:daily:dispatch:group | 下发指导组 |
| violation:daily:confirm:group | 指导组确认 |
| violation:daily:feedback | 不属实反馈 |
| violation:daily:recheck | 复核 |
| violation:daily:final | 主任最终确认 |
| violation:daily:result | 结果库查询 |
| violation:daily:export | Excel 导出 |
| violation:daily:log | 查看业务日志 |
| violation:daily:attachment:view | 查看附件 |
| violation:daily:attachment:download | 下载附件 |
| violation:daily:share | 分享业务卡片 |

### 14.3 数据权限

必须按角色和组织关系过滤数据。

禁止仅根据前端传参过滤。

后端查询必须自动注入当前用户数据范围。

### 14.4 字段权限

关键字段如违标编码、责任部门、工号、责任人、违章日期、拟考核内容，在不同状态下可编辑权限不同。

例如：

- 草稿：分析员可编辑。
- 班长待审核：分析员不可编辑，班长可退回。
- 主任待审核：班长和分析员不可编辑，主任可退回或处理。
- 已入结果库：默认不可编辑，只能走修订流程。

## 15. 导出规则

### 15.1 通用导出规则

- 所有列表页原则上支持导出。
- 导出权限独立于查看权限。
- 所有导出必须由后端校验权限。
- 所有导出必须写导出日志。
- 导出日志记录导出人、时间、条件、数量、文件名。

### 15.2 本模块特殊导出规则

- 导出标题包含提报日期。
- 表格列只导出“违章发生日期”。
- 不导出分数、扣款字段。
- 导出表格应接近现有 Excel 公示格式。
- 支持按状态、日期、责任部门、人员、编码筛选后导出。

### 16.1 必须记录的业务动作

### 16.2 日志字段

### 16.4 入结果库前检查

- 是否有主任最终确认日志。
- 若存在反馈，是否有反馈处理日志。
- 若存在附件，附件是否绑定业务记录和动作日志。

没有完整日志证据链的数据，不允许进入结果库。

## 17. 结果库规则

### 17.1 入库条件

只有主任最终确认的数据才能进入结果库。

入库前必须满足：

- 当前状态为 `FINAL_CONFIRMED`。
- 当前用户具备 `violation:daily:final` 或结果入库权限。
- 业务日志链路完整。
- 必填快照完整。

### 17.2 结果快照

入库时必须保存完整快照，包括：

- 记录基本信息
- 人员快照
- 组织快照
- 违标编码快照
- 审核流程摘要
- 附件摘要
- 反馈处理摘要

### 17.3 版本规则

结果库不允许直接覆盖历史。

如需修订：

- 新增版本。
- 记录修订原因。
- 记录修订人和时间。
- 原版本保留。

### 17.4 撤销不计入

认定不属实或撤销时：

- 原记录保留。
- 结果库记录可标记为 `cancelled`。
- 统计时默认不计入。
- 日志必须完整。

## 23. 验收清单

开发完成后，至少验证：

- 分析员只能看到本人数据。
- 班长只能审核所属分析员数据。
- 主任能处理分析室范围数据。
- 车间只能看到下发到本车间的数据。
- 车队只能看到下发到本车队的数据。
- 指导组只能看到本指导组待确认数据。
- 未授权用户无法通过 URL 直接打开记录。
- 未授权用户无法导出数据。
- 违标编码不存在时不能提交。
- 校验失败的导入行不能提交。
- 退回必须填写原因。
- 不属实反馈必须填写原因并可上传附件。
- 所有流程动作都有业务日志。
- 入结果库前有完整日志证据链。
- 结果库保留快照和版本。
- 已撤销不计入数据不参与默认统计。
