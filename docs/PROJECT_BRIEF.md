# 项目背景说明

## 项目名称

综合分析室数据分析平台

## 项目定位

本项目是内部中后台管理平台，不是单一页面 Demo。平台后续会承载多个分析、统计、审核、导入导出和结果库模块。

## 一级功能规划

- 功能首页
- 信箱中心
- 违章管理
- LKJ 分析
- 音视频分析
- 单兵分析
- 解锁统计
- 调度命令
- 施工作业
- 试运机车
- 月度报表
- 统计分析
- 基础数据
- 系统管理

## 第一阶段样板功能

第一阶段只做：

- 违章管理 > 每日 LKJ 音视频违标公示

该功能作为平台样板业务，用于沉淀公共能力：权限、信箱、待办、流程、附件、日志、导入导出、结果库。

## 核心业务流程

```text
分析员录入或 Excel 导入
→ 违标编码校验
→ 表格预览
→ 确定提交
→ 班长审核
→ 分析室主任审核
→ 下发责任车间
→ 车队
→ 指导组确认
→ 指导组确认无误或反馈不属实
→ 逐级返回
→ 分析室复核
→ 主任最终确认
→ 进入 LKJ/音视频违标结果数据库
```

## 核心规则

1. 登录后进入极简功能首页，只显示功能名称方块。
2. 违章管理只是其中一个模块，不能把整个平台做成单一违章系统。
3. 信箱中心是主要流转入口。
4. 录入界面要精简。
5. 日期分为提报日期和违章日期。
6. Excel 导出标题带提报日期，表格列只导出违章发生日期。
7. 违标编码必须来自数据库，不存在不允许录入。
8. 编码、性质、类别、类型必须强校验。
9. 指导组可以反馈不属实，必须填写理由并上传证据附件。
10. 所有业务动作必须写业务证据链日志。
