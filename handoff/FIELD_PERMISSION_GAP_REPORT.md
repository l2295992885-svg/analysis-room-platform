# 字段权限缺口报告

更新时间：2026-06-29
阶段：阶段 14 上线前安全边界补强与工作区整理

## 1. 结论

每日 LKJ 已接入字段权限第一版。当前后端不再只依赖前端禁用字段：草稿状态允许编辑录入字段，非草稿状态禁止直接修改业务事实字段；审核、下发、反馈等流程字段通过专用动作写入；编码、性质、类别、类型不一致时需要 `violation:daily:override`，授权覆盖会写业务日志。

本轮端到端验收已覆盖“非 DRAFT 修改事实字段被拒绝”，脚本返回 403 并通过。结果库更正也走专用接口追加版本，不直接覆盖历史。

这仍不是最终字段权限平台。当前实现是每日 LKJ 服务层内的清晰校验函数，尚未抽成全平台可配置字段权限引擎。

## 2. 已接入规则

| 场景 | 当前规则 | 状态 |
| --- | --- | --- |
| DRAFT 编辑 | 允许编辑录入字段并重新校验违标编码和人员快照 | 已接入 |
| 非 DRAFT 编辑事实字段 | `updateRecord(...)` 拒绝，返回 403 或业务错误 | 已接入 |
| 审核阶段 | 通过班长/主任审核动作写意见或退回原因，不开放事实字段编辑 | 已接入 |
| 下发阶段 | 通过下发动作写下发对象和意见，并校验动作角色范围 | 已接入 |
| 指导组反馈 | 通过反馈动作写原因类型、说明，可上传证据附件 | 已接入 |
| 结果库阶段 | 结果库默认只读；更正必须走更正接口追加新版本 | 已接入 |
| 编码元数据覆盖 | 无 `violation:daily:override` 时阻止覆盖；有权限时写 `CODE_METADATA_OVERRIDE` 日志 | 已接入 |
| 导入确认 | 导入提交时校验编码元数据覆盖权限 | 已接入 |

## 3. 当前字段分组

| 字段组 | 示例字段 | 当前处理 |
| --- | --- | --- |
| 录入事实字段 | 违章日期、时间、责任部门、工号、责任人、机车、车次、地点、拟考核内容 | 仅 DRAFT 编辑 |
| 编码派生字段 | 违标编码、性质、类别、类型 | 必须来自编码库；覆盖需授权 |
| 审核字段 | 审核意见、退回原因 | 仅审核动作写入 |
| 下发字段 | 车间、车队、指导组、下发意见 | 仅下发动作写入 |
| 反馈字段 | 不属实原因、说明、证据附件 | 仅指导组反馈和附件上传动作写入 |
| 结果字段 | included、resultStatus、correctReason、resultSnapshot | 仅入库/撤销/更正动作写入 |

## 4. 本轮自动验收结果

| 验收项 | 结果 |
| --- | --- |
| 非 DRAFT 修改事实字段 | 返回 403，脚本通过 |
| 编码覆盖权限 | 服务层已接入 `violation:daily:override` 校验 |
| 指导组反馈 | 必须走反馈动作并写流程日志，脚本通过 |
| 结果库直接覆盖历史 | 不允许；更正追加新版本，脚本验证版本 1/2 保留 |

## 5. 仍需补强的问题

| 缺口 | 说明 |
| --- | --- |
| 平台化字段权限服务 | 当前规则在每日 LKJ 服务内，后续模块复用需要抽象 |
| 字段级配置 | 尚未支持数据库配置“可编辑/只读/不可见/需要覆盖授权” |
| 前端字段状态联动 | 前端已有基础禁用，但可继续根据后端字段权限结果动态渲染 |
| 结构化字段变更审计 | 当前关键覆盖写日志，后续可统一字段名、旧值、新值、原因、授权人 |
| 更细的覆盖授权 | 当前覆盖权限较粗，生产可按编码、人员、部门等字段拆细 |

## 6. 后续验收点

| 验收项 | 期望 |
| --- | --- |
| 非 DRAFT 修改录入字段 | 返回 403 或业务错误 |
| 编码与性质不一致且无覆盖权限 | 阻止提交 |
| 编码与性质不一致且有覆盖权限 | 允许并记录授权覆盖日志 |
| 指导组反馈缺少原因 | 阻止提交 |
| 结果库直接覆盖历史 | 不允许 |
| 结果库更正 | 追加新版本并保留原版本 |
