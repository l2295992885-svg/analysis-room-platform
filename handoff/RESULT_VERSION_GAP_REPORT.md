# 结果库版本能力缺口报告

更新时间：2026-06-29
阶段：阶段 14 上线前安全边界补强与工作区整理

## 1. 结论

每日 LKJ 结果库版本能力已从“只有入库快照和版本字段”推进到第一版可用：已补齐结果详情、版本列表、版本对比、结果更正接口和前端入口。更正必须填写原因，并通过追加新版本实现，不覆盖旧版本。

本轮端到端验收已验证：入库生成版本 1，结果更正追加版本 2，旧版本保留，版本对比接口可用，无更正权限返回 403。

当前仍不是最终审计版本能力。版本导出、复杂字段差异展示、批量更正、审批型更正流程和更细的字段级更正权限仍未实现。

## 2. 已实现能力

| 能力 | 当前实现 |
| --- | --- |
| 结果详情 | `GET /violation/daily/results/{resultId}` |
| 版本列表 | `GET /violation/daily/results/{resultId}/versions` |
| 版本对比 | `GET /violation/daily/results/{resultId}/versions/compare` |
| 结果更正 | `POST /violation/daily/results/{resultId}/correct` |
| 追加版本 | 更正调用 `insertResult(...)`，基于 `maxVersion + 1` 追加 |
| 保留历史 | 旧版本不更新、不删除 |
| 更正原因 | 更正 BO 要求填写 `correctReason` |
| 权限校验 | 更正需要 `violation:daily:result:correct`，版本查看需要 `violation:daily:result:version:view` |
| 数据权限 | 结果详情、版本、对比、更正均回查来源记录并校验记录级范围 |
| 审计 | 更正写操作日志和每日 LKJ 业务流程日志 |
| 前端入口 | 结果库列表新增详情、版本、对比、更正入口 |

## 3. 当前表结构基础

| 字段 | 用途 |
| --- | --- |
| `result_id` | 结果库记录 ID |
| `record_id` | 来源每日违标记录 |
| `batch_id` | 来源导入批次 |
| `result_version` | 结果版本号 |
| `result_status` | 结果状态，例如 `ARCHIVED`、`CANCELLED_EXCLUDED` |
| `included` | 是否计入统计 |
| `result_snapshot` | 入库或更正时业务快照 JSON |
| `archived_by`、`archived_user_name`、`archived_time` | 入库/更正操作人和时间 |
| `corrected_from_result_id` | 更正来源结果 ID |
| `correct_reason` | 更正或撤销原因 |
| `del_flag` | 逻辑删除标记 |

## 4. 本轮自动验收结果

| 验收项 | 结果 |
| --- | --- |
| 入库版本 1 | 脚本通过 |
| 更正版本 2 | 脚本通过 |
| 旧版本保留 | 脚本通过 |
| 版本对比 | HTTP 200，脚本通过 |
| 无权限更正 | 返回 403，脚本通过 |
| 结果库导出 | 受导出权限和数据权限共同约束，脚本通过 |

## 5. 仍需补强的问题

| 缺口 | 说明 |
| --- | --- |
| 版本导出 | 尚未实现单条结果版本历史导出 |
| 字段差异展示 | 当前对比为基础快照差异，后续可细化中文字段名和业务语义 |
| 更正审批 | 当前更正是有权限即追加版本，生产可考虑更正审批流 |
| 批量更正 | 尚未实现批量更正或批量撤销 |
| 更正权限细分 | 生产可按字段类型、结果状态、组织范围进一步拆分更正权限 |
