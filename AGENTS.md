# AGENTS.md

本文件是 Codex / AI 编程工具在本仓库工作时必须遵守的项目规则。

## 项目定位

本项目为“综合分析室数据分析平台”，属于内部中后台管理系统。第一阶段只实现“违章管理 > 每日 LKJ 音视频违标公示”完整闭环，但底层结构必须按平台化设计，后续要能扩展 LKJ 分析、音视频分析、单兵分析、解锁统计、调度命令、施工作业、试运机车、月度报表、统计分析等模块。

## 技术路线

- 前端参考 Art Design Pro 的 UI 风格、布局、表格、表单、搜索栏、主题和常用组件。
- 后端使用 Spring Boot。
- 第一版数据库使用 MySQL 8.x。
- 权限模型参考若依成熟设计：用户、角色、部门、菜单、按钮权限、数据权限、登录日志、操作日志、安全拦截。
- 第一版不使用微服务，不引入复杂工作流引擎，不做大屏。

## 禁止事项

1. 不允许重写登录认证核心，除非用户明确要求。
2. 不允许自己发明一套新的用户权限系统。
3. 不允许只在前端隐藏按钮来代替权限控制。
4. 不允许业务接口绕过后端权限校验。
5. 不允许业务查询绕过数据权限过滤。
6. 不允许导出未授权数据。
7. 不允许物理删除历史业务数据。
8. 不允许覆盖历史违标编码快照。
9. 不允许把单个业务模块写死成不可复用结构。
10. 不允许在没有日志证据链的情况下让数据进入最终结果库。

## 必须事项

1. 所有新增接口必须设计权限标识。
2. 所有查询必须考虑数据权限过滤。
3. 所有导出必须校验导出权限并写导出日志。
4. 所有提交、审核、退回、修改、反馈、附件上传、入库操作必须写业务证据链日志。
5. 所有附件必须绑定业务记录、上传人、上传时间、业务动作。
6. 历史违章记录必须保存编码快照、人员快照、组织快照。
7. 新模块必须遵守统一目录、命名、权限、日志、导入导出规则。
8. 每次修改代码后必须说明新增/修改文件、影响范围、测试结果和风险点。

## 第一阶段业务范围

仅实现：

- 违章管理 > 每日 LKJ 音视频违标公示

核心流程：

```text
分析员录入/Excel导入
→ 编码校验
→ 表格预览
→ 提交班长
→ 班长审核
→ 主任审核
→ 下发车间
→ 车队
→ 指导组确认/反馈不属实
→ 逐级返回
→ 分析室复核
→ 主任最终确认
→ 入结果库
```

## 推荐分支

- `feature/init-scaffold`：项目骨架
- `feature/system-permission`：用户、角色、部门、菜单、按钮、数据权限
- `feature/business-common`：信箱、待办、附件、业务日志、导入导出
- `feature/daily-violation`：每日 LKJ 音视频违标公示
- `feature/import-export`：Excel 导入导出模板
- `feature/security-review`：权限和安全审查

## 前端工程规则

- 前端工程位于 `frontend/`。
- 前端基础模板来源于 Art Design Pro。
- 后续前端开发必须遵守 Art Design Pro 的目录结构和工程约定。
- 前端技术路线以 Vue3、TypeScript、Vite、Element Plus、pnpm 工作流为基础。
- 不得保留模板演示页面作为正式业务功能。
- 不得只做前端按钮隐藏，后端接口仍必须做权限校验。
- 前端页面应服务于综合分析室平台业务，包括功能首页、信箱中心、待办中心、文件中心、聊天协同、违章管理、每日 LKJ 音视频违标公示、基础数据、系统管理、统计分析等模块。
